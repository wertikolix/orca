# 0.8.2

## New features

- **Configurable superscript/subscript parsing**: `OrcaMarkdownParser` now accepts `enableSuperscript` and `enableSubscript` parameters (both default to `true`). Set either to `false` to disable `^text^` or `~text~` parsing and keep the raw delimiters as plain text.

```kotlin
val parser = OrcaMarkdownParser(
    enableSubscript = false,  // ~text~ stays as plain text
)
```

## Fixes

- **Release CI**: Maven Central staging publish step no longer fails the workflow when the Sonatype API returns a transient error. Added retry logic (3 attempts, 10s delay) and graceful fallback with a warning instead of hard failure.

## Notes

- No breaking changes.
- Default behavior is identical to 0.8.1.
- Strikethrough (`~~text~~`) is unaffected by the subscript toggle -- it is handled at a different parsing layer.
