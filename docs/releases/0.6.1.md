# Orca 0.6.1

Release focus: parser API naming cleanup and publishing pipeline stability.

## Highlights

- **New primary parser name:** `OrcaMarkdownParser`.
- **Compatibility kept:** `IntellijMarkdownOrcaParser` remains available as a deprecated alias.
- **No parser behavior changes:** backend is still `org.jetbrains:markdown` with `GFMFlavourDescriptor`.
- **Maven Central publish hardening:**
  - CI signing supports in-memory keys (`signingInMemoryKey`).
  - Per-publication `javadoc` artifacts are generated for Central validation.

## Migration from 0.6.0

```diff
- import ru.wertik.orca.core.IntellijMarkdownOrcaParser
+ import ru.wertik.orca.core.OrcaMarkdownParser

- val parser: OrcaParser = IntellijMarkdownOrcaParser()
+ val parser: OrcaParser = OrcaMarkdownParser()
```

`IntellijMarkdownOrcaParser` is still supported for backward compatibility, but will be removed in a future major release.
