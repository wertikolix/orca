# Orca 0.4.3

Release focus: HTML handling and front matter parsing fixes.

## Fixes

- Fixed HTML entity decoding order: `&amp;` is now decoded first, preventing double-decoding of escaped entities (e.g. `&amp;lt;` correctly becomes `&lt;` instead of `<`).
  - Affected: core parser, inline text renderer, block node renderer.
- Fixed HTML tag stripping regex: now requires valid tag name start (`</?[a-zA-Z]...>`), preventing false matches on content like `<3` or `<hello`.
  - Affected: core parser, inline text renderer, block node renderer.
- Fixed front matter value parsing: quotes are now only stripped when they form matching pairs (`"..."` or `'...'`), preserving intentionally quoted content like `"'value'"`.

## No breaking changes.
