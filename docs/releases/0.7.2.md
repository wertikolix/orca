# 0.7.2

Test coverage release.

## Added

### OrcaMarkdownParserTest
- `parserRequiresPositiveCacheSize` — validates `cacheSize > 0` constructor guard
- `clearCacheInvalidatesStoredEntries` — verifies `clearCache()` forces re-parse
- `parseCachedWithDiagnosticsReturnsSameResultForSameKeyAndInput` — cache hit
- `parseCachedWithDiagnosticsInvalidatesOnInputChange` — cache invalidation
- `parseWhitespaceOnlyInputProducesEmptyDocument` — whitespace-only input edge case
- `parseSetextHeading` — setext-style headings (`===` / `---`) were untested
- `parseIndentedCodeBlock` — 4-space indented code blocks
- `parseAutolinkAngleBracketSyntax` — `<https://...>` autolink syntax
- `parseFullReferenceLinkResolvesDestination` — `[text][ref]` reference links
- `parseNestedOrderedList` — nested ordered list hierarchy
- `parseImageWithNoAltText` — image with empty alt
- `parseMixedBoldItalicNesting` — `*bold italic*` nesting

### OrcaParserTest
- `defaultParseCachedWithDiagnosticsDoesNotCache` — default `OrcaParser` interface does not cache
- `defaultParseCachedDoesNotCache` — same for `parseCached`

### OrcaStreamingParserTest
- `cacheSizeEvictsOldestEntryWhenFull` — LRU eviction when cache is at capacity

### MarkdownUrlSafetyTest
- Unit tests for URL safety security functions

### Front matter extraction
- Unit tests for front matter extraction edge cases
- Fix: illegal characters in test function name
- Fix: rewrite front matter and footnote tests via public API
