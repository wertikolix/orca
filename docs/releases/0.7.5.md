# 0.7.5

Syntax highlighting expansion + test coverage release.

## Fixed

- **Orphaned test methods in `OrcaCodeBlockTest`** — 11 `@Test` functions were defined outside the class body as top-level functions. They compiled but were never discovered or run by the test runner. Moved inside the class.
- **SQL keyword matching is now case-insensitive** — `SELECT`, `FROM`, `WHERE` etc. are highlighted correctly (previously only lowercase matched).

## Added

### Syntax highlighting for new languages
- **Python**: `def`, `class`, `import`, `return`, `None`, `True`, `False`, `and`/`or`/`not`, etc.
- **Go**: `func`, `var`, `const`, `struct`, `interface`, `nil`, `defer`, `select`, etc.
- **Rust**: `fn`, `let`, `mut`, `impl`, `trait`, `match`, `None`, `Some`, `Ok`, `Err`, etc.
- **C/C++**: `int`, `float`, `void`, `struct`, `typedef`, `extern`, `NULL`, etc.
- **Swift**: `func`, `let`, `var`, `protocol`, `guard`, `nil`, `throw`, etc.
- **C#**: `class`, `namespace`, `using`, `foreach`, `async`/`await`, etc.
- Java and JS/TS were already present.

### OrcaInlineParserTest (new file, 33 tests)
- Bold/italic nesting, strikethrough with bold
- Inline code backtick escaping, space stripping
- Link with title, empty destination, code in link content
- Short reference link resolution
- Image with blank alt (null), image with title
- HTML inline nodes, multiple inline footnotes
- Inline footnote with formatting, mixed ref+inline footnotes
- GFM autolink without scheme, code fence edge cases
- Heading level 6, heading with inline formatting
- Soft line break, nested block quote, list with asterisk bullets
- Ordered list `startNumber`, list item with multiple paragraphs

### OrcaHtmlUtilsTest (new file, 22 tests)
- `decodeBasicHtmlEntities`: all 6 entities, multiple in one string, empty, no entities
- `htmlBlockFallbackText`: tag stripping, `br`→`\n`, block break tags, entity decode, trim, tags-only
- `htmlInlineFallbackText`: strip+decode, `br`→`\n`, empty, plain text

### OrcaFrontMatterEdgeCasesTest + OrcaFootnoteEdgeCasesTest (new file, 18 tests)
- YAML with `...` closing delimiter
- TOML with quoted values, YAML with quoted values
- Front matter ignores comment lines (`#`), list items (`- `)
- Empty body, no closing delimiter (ignored), raw doesn't include delimiters
- Front matter not conflicting with thematic break `---`
- Footnote with multi-paragraph body, empty body, label trimming
- Multiple refs to same label, inline footnote with nested brackets
- Footnote reference in heading, footnote with formatted body

### Syntax highlight test coverage
- 9 new tests for all new languages in `OrcaCodeBlockTest`
