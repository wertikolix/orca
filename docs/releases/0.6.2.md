# Orca 0.6.2

Release focus: parser API naming cleanup + CI publishing reliability.

## Highlights

- **Primary parser API:** `OrcaMarkdownParser`.
- **Compatibility alias kept:** `IntellijMarkdownOrcaParser` remains available and deprecated.
- **No parser behavior changes:** backend remains `org.jetbrains:markdown` with `GFMFlavourDescriptor`.
- **Maven Central publish reliability:**
  - CI signing key normalization now handles multiline, escaped `\n`, and base64-encoded secrets.
  - Both Gradle-property and direct env paths are populated for in-memory signing.
  - Per-publication `javadoc` artifacts are attached for Central validation.

## Migration from 0.6.0

```diff
- import ru.wertik.orca.core.IntellijMarkdownOrcaParser
+ import ru.wertik.orca.core.OrcaMarkdownParser

- val parser: OrcaParser = IntellijMarkdownOrcaParser()
+ val parser: OrcaParser = OrcaMarkdownParser()
```

`IntellijMarkdownOrcaParser` stays as a backward-compatible alias for now.
