# FhMd 0.1.0-alpha03

Release focus: infrastructure hardening and release pipeline hygiene.  
Engine scope is unchanged.

## Highlights

- Added non-blocking Gradle deprecation scan in CI:
  - `--warning-mode all` run
  - log uploaded as workflow artifact
- Added report artifact upload on CI failures for faster debugging.
- Added dedicated `release.yml` workflow for version tags (`0.*`):
  - runs tests
  - builds `assembleRelease` + `bundleRelease`
  - uploads APK/AAB artifacts to both workflow artifacts and GitHub Release
- Kept JNI debug symbols retention only in `debug` build type to avoid release-size impact.

## Supported (unchanged)

- Parser backend: `commonmark-java` (`0.27.1`)
- Public parser API:
  - `interface FhMdParser { fun parse(input: String): FhMdDocument }`
- Blocks:
  - heading
  - paragraph
  - bullet list / ordered list
  - quote
  - fenced code block / indented code block
- Inlines:
  - text
  - bold
  - italic
  - inline code
  - link
  - soft/hard line breaks
- Compose renderer:
  - `FhMd(markdown = ...)`
  - `FhMd(document = ...)`
  - `onLinkClick` callback
  - baseline styling via `FhMdStyle`

## Not Supported Yet (unchanged)

- tables
- images
- HTML blocks/inline HTML
- strikethrough
- task lists
- thematic breaks (`---`)
- footnotes/front matter/extensions
- non-Android targets (multiplatform-ready architecture only)

## Verification

```bash
./gradlew --no-daemon --build-cache :fhmd-core:test :fhmd-compose-android:testDebugUnitTest :sample-app:assembleDebug
./gradlew --no-daemon --build-cache :sample-app:assembleRelease :sample-app:bundleRelease
```
